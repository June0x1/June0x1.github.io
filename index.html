Git Tips

1-Git Create Repository

1.1-Git Init

git init (git init newrepo)
git add .
git add README.md
git commit -m "commit info"

1.2-Git Clone

git clone <repo>
git clone <repo> <directory>

For example:

git clone git://github.com/schacon/grit.git
git clone git://github.com/schacon/grit.git mygrit

1.3-Git Config

Show git config information:

git config --list
credential.helper=osxkeychain
core.repositoryformatversion=0
core.filemode=true
core.bare=false
core.logallrefupdates=true
core.ignorecase=true
core.precomposeunicode=true

Edit git config file:

git config -e    # Junst for this repository
git config -e --global   # for all repository

Config user information:

git config --global user.name "xxxx"
git config --global user.email xxxx@xxxx.com

2-Git Operations

git init      # init repository
git clone     # clone a remote repository  
git add       # add file to staging area
git status    # view the repository status, show the changed file
git diff      # view the difference of stating area and workspace
git commit    # upload stating area to local repository
git reset     # rollback the version
git rm        # delete file from stating area and workspace
git mv        # remove or rename workspace file
git log       # view the history logs
git blame <file> #view the histroy logs from blame
git remote    # remote option
git fetch     # download code from remote repository
git pull      # download and merge code from remote repository 
git push      # upload and merge code to remote repository

3-Git lfs install and upload step

Git lfs is a tool for upload big file to github. install step as follow:

curl -s https://packagecloud.io/install/repositories/github/git-lfs/script.deb.sh | sudo bash
sudo apt-get install git-lfs
git lfs install

upload step:

git lfs track "file"
git add .gitattributes
git commit -m "submit file"
git push -u origin main 

git add file
git commit -m "add file"
git push -u origin main

4-Git Rollback Version

Rollback version:

git log -10
git reset --hard xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
git pull origin main
git merge dev

How to create a hexo blog using Github page

This is the first post about Hexo blog. There are some steps about how to create a bolg using GitHub Page.

Quick Start

Install git, nodejs, npm, hexo

sudo apt-get install git
sudo apt-get install npm
sudo apt-get install nodejs

If the nodejs version is below 18.x, please update it.

curl -sL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

Install hexo

npm install hexo -g

Test hexo version

hexo -v

Install hexo git deployer tools

npm install --save hexo-deployer-git

Github config

Generate ssh key

ssh-keygen -t rsa -C "mailaddress@xx.com"

Create a new ssh-key, then put the key(in the /home/xx/.ssh/id_rsa.pub) into the ssh-rsa value.

Test it

ssh -T git@github.com

Config the github account

git config --global user.name "username" 
git config --global user.email "username@xx.com"

Create a new post

Start the hello world page

hexo init  #init the blog

hexo g     #generate the static pages

hexo s     #start the local site http://localhost:4000/

Update the config, please edit the _config.yml.

Upload to Github

Create a new respository, name is username.github.io.

Config file(_config.yml), update deploy section as follow.

deploy:
  type: git
  repository: git@github.com:username/username.github.io.git
  branch: main

upload to Github

hexo clean
hexo g
hexo d

Some hexo command

hexo new "postName"
hexo new page "pageName"
hexo generate
hexo server
hexo deploy
hexo help
hexo version
